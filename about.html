Overview

The world is moving towards becoming entirely driven by technology. Building software for billions of people is hard. With the emergence of Cloud computing we've accelerated software delivery and unlocked elasticity in infrastructure management but software development tools have largely lagged behind. At Micro we've developed a model for effectively leveraging the Cloud and now want to share that with the world.

Design

Micro services is a software design pattern which accelerates development by breaking down applications into a number of smaller services which can be developed, deployed and scaled independently.


Philosophy
  • - Do one thing well - Services follow the unix philosophy and focus on solving one problem
  • - Embrace failure - The culture and design principles should follow anti-fragility
  • - Automate everything - Automation, fast iteration and testing need to be part of the system

Architecture

A Micro services architecture enables products to be broken down into smaller services which teams work on and scale independently of each other. Each team became responsible for the delivery of their particular service which others come to use through common APIs.


Benefits
  • - Autonomous organisation - teams self organise around different products and services
  • - Increased fault tolerance - develop, deploy and manage services independently
  • - Speed of execution - reuse services through APIs rather than building from scratch

Technology

Micro is a cloud native development platform. We want to empower developers to build software at an accelerated pace from early stage prototyping all the way through to large scale production deployment without having to think about managing the infrastructure. We aspire to become host to the next generation of cloud services.
